# Screenshot Guide

This document explains which screenshots to capture for the portfolio README.

## 1. Start and Status

Capture:
- the bot greeting after `/start`
- the `/status` response showing an empty draft

Purpose:
- shows the Telegram entry point and the revision workflow

## 2. Image Ingest

Capture:
- sending a COMP9414 question PNG
- the bot reply confirming the image was queued for extraction

Purpose:
- demonstrates multi-modal input handling

## 3. Solution Notes

Capture:
- sending your own solution text after the image
- the bot reply confirming the text was added to the draft

Purpose:
- shows the "same conversation, multiple steps" workflow

## 4. Save Result

Capture:
- running `/save`
- the bot reply with the saved Markdown file path

Purpose:
- proves the knowledge-base output is actually produced

## 5. Git History

Capture:
- `git log --oneline -- knowledge-base/`

Purpose:
- shows the Git-backed knowledge base aspect of the project
